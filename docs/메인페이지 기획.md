## 💻 연애판별소(연판소) 메인 페이지 핵심 구성안

### 1. Hero Section (첫인상 및 후킹)

- **목적**: 페이지 접속 후 3초 이내에 20대 여성의 공감을 이끌어내고 이탈을 막음.
    
- **주요 내용**:
    
    - 감성적이면서도 직관적인 카피: _"이 썸, 계속 가야 할까? 아니면 지금 손절해야 할까?"_, _"시간 낭비, 감정 낭비 없는 확신 가득한 연애를 위해"_
        
    - **시각 요소**: 밤하늘의 별자리나 모던한 명리 그래픽(네온/그라데이션 효과)이 은은하게 움직이는 모던 그래픽 백그라운드.
        
    - **CTA 버튼**: `[내 썸 상대 판별해보기 🔍]` (마우스 호버 시 빛나는 글래스모피즘 스타일)
        

### 2. Problem & Empathy Section (공감 및 페인 포인트 자극)

- **목적**: 사용자가 겪고 있는 답답한 상황을 시각적으로 나열하여 "이거 완전 내 얘기인데?"라는 공감을 유도.
    
- **주요 내용**: 20대 여성이 썸 탈 때 겪는 스트레스 체크리스트.
    
    - _상대가 선톡은 하는데 만나자는 말은 안 해요._
        
    - _사귀는 것도 아니고, 끝난 것도 아닌 이 애매한 관계..._
        
    - _나 혼자만 안달 나고 자존감 깎이는 기분이에요._
        
- **시각 요소**: 스마트폰 메신저 대화창 UI(상대의 애매한 답변 예시)를 CSS 카드로 구현, 스크롤 시 부드럽게 페이드인(Fade-in)되는 애니메이션.
    

### 3. Solution Section (연판소의 해결책 제시)

- **목적**: 사주 분석을 기반으로 한 '썸 손절 판별 리포트'가 이 문제를 어떻게 깔끔하게 해결해 주는지 논리적으로 제시.
    
- **주요 내용**: 데이터 기반 사주 분석의 신뢰성과 '연판소'만의 차별점 설명.
    
    - _단순한 타로가 아닙니다. 두 사람의 타고난 기운과 명리학적 데이터를 분석한 객관적 지표._
        
- **시각 요소**: 톱니바퀴나 데이터가 분석되는 듯한 정교한 인라인 SVG 아이콘, 깔끔한 3단 그리드(Grid) 레이아웃.
    

### 4. Report Preview Section (리포트 미리보기 - 핵심 핵심!)

- **목적**: 사용자가 실제로 받게 될 리포트의 퀄리티를 미리 보여주어 구매 욕구를 자극. (UX 디자인의 'Show, Don't Tell' 원칙)
    
- **주요 내용**: 제공되는 5대 핵심 콘텐츠의 가상 샘플 대시보드.
    
    - 상대의 연애 성향 / 현재 관계 흐름 / 관계 발전 가능성 / 위험 신호 / 행동 가이드
        
- **시각 요소**: **Chart.js를 활용한 '관계 지속 지수' 및 '손절 필요도' 동적 차트(원형 또는 방사형 그래프)** 시각화. '위험 신호(Red Flags)' 레이블에 경고 아이콘 삽입.
    

### 5. Benefit Section (기대 효과)

- **목적**: 리포트를 구매했을 때 사용자가 얻을 수 있는 심리적, 시간적 이득을 확실히 인지시킴.
    
- **주요 내용**:
    
    - _감정 낭비 감소 & 자존감 보호_: 혼자 의미 부여하며 상처받는 시간 끝!
        
    - _빠른 의사결정 & 미래 리스크 예방_: 몇 개월 뒤 후회할 똥차, 미리 걸러내기.
        
- **시각 요소**: 깔끔한 보더 라운드와 소프트 섀도우가 적용된 카드 뉴스 형태의 레이아웃.
    

### 6. Social Proof & Reviews (실제 고객 후기)

- **목적**: 먼저 서비스를 이용한 사람들의 생생한 후기를 통해 신뢰도(Social Proof)를 확보.
    
- **주요 내용**: "연판소 덕분에 3개월 질질 끌던 똥차 걷어차고 새 출발 했어요!", "리포트 보고 마음 정리했는데, 진짜 소름 돋게 맞추네요" 등의 바이럴 후기.
    
- **시각 요소**: 20대 타깃에 친숙한 인스타그램 피드 스타일 혹은 슬라이더(Carousel) 컴포넌트 구조.
    

### 7. Final Call to Action (최종 구매 유도 및 푸터)

- **목적**: 페이지를 끝까지 읽은 사용자가 고민 없이 바로 분석을 시작하도록 유도.
    
- **주요 내용**:
    
    - _"당신의 시간과 감정은 썸 상대보다 훨씬 더 소중합니다. 지금 판별해 보세요."_
        
    - 가격 정책 및 즉시 분석 가능 문구 유치.
        
- **시각 요소**: 화면을 가득 채우는 화려한 그라데이션 배경과 시선을 사로잡는 대형 구매 버튼.
    

## 🎨 디자인 콘셉트 및 무드 제안 (UI/UX)

- **컬러 스키마**:
    
    - **Main**: 미드나잇 퍼플 (Deep Midnight Purple - 사주/명리의 신비롭고 전문적인 느낌)

## 📊 2단계: 기능 스펙 및 정보 구조도 (IA) 설계

연판소 메인 페이지는 단순한 랜딩 페이지를 넘어, 사용자의 입력을 유도하고 리포트의 가치를 시각적으로 증명하는 '인터랙티브 웹 앱'의 성격을 가집니다.

### 1. 정보 구조도 (Information Architecture)

- **Main Page (index.html)**
    
    - **Header**: 로고, 메뉴 링킹(서비스 소개, 리포트 샘플, 후기), `[지금 판별하기]` 빠른 버튼
        
    - **Hero**: 타이틀 카피, 메인 그래픽 컴포넌트, 즉시 진입 CTA
        
    - **Problem Area**: 3대 페인 포인트 카드 (아코디언 또는 탭 구조)
        
    - **Solution Area**: 사주 기반 분석 프로세스 (Step 1 -> Step 2 -> Step 3 모던 타임라인)
        
    - **Report Preview Area (핵심 기능)**: 가상 데이터 기반의 인터랙티브 대시보드 (차트 포함)
        
    - **Social Proof Area**: 평점 및 롤링 텍스트/카드가 포함된 후기 슬라이더
        
    - **Conversion (CTA) Area**: 사주 입력 폼 (이름, 생년월일시, 상대방 정보 간단 입력 인터페이스)
        
    - **Footer**: 이용약관, 개인정보처리방침, CS 안내
        

### 2. 필수 기능 스펙 및 시각화 데이터 정의

안티그래비티가 단순 텍스트가 아닌 라이브러리와 CSS를 조합해 빌드해야 할 핵심 스펙입니다.

- **인터랙티브 리포트 미리보기 (Chart.js 연동)**
    
    - _데이터 정의_: 사용자가 마우스를 올리거나 탭을 전환할 때, '매칭 지수(%)', '손절 지수(%)', '소통 성향' 등이 동적으로 차트로 그려짐.
        
    - _시각화_: ApexCharts 또는 Chart.js의 레이더 차트(Radar Chart)와 도넛 차트(Doughnut Chart)를 활용해 관계의 지속 가능성을 시각화.
        
- **실시간 카운트다운 및 알림 애니메이션**
    
    - _기능_: 메인 진입 시 "오늘 이미 2,341명의 여성이 썸을 판별했습니다"라는 알림 팝업이 좌측 하단에 부드럽게 나타났다가 사라짐 (신뢰도 부여).
        
- **입력 폼 컴포넌트 (Validation 및 UI)**
    
    - _기능_: 생년월일시 입력 시 모던한 드롭다운 또는 깔끔한 데이트 피커(Date Picker) UI 적용. 필수 항목 누락 시 테일윈드 쉐이크(Shake) 애니메이션과 레드 보더 경고 레이아웃 작동.
        

## 🔍 기획자 제안: 누락 방지를 위한 예외 처리 및 추가 제안

1. **모바일 퍼스트 UX**: 20대 여성 타깃의 90% 이상은 인스타그램 광고 등을 통해 모바일로 접속합니다. 따라서 모든 레이아웃은 모바일 화면에서 1열로 깔끔하게 떨어지는 반응형 그리드(Responsive Grid)가 필수적입니다.
    
2. **로딩 인터랙션**: 사용자가 사주 정보를 입력하고 `[판별하기]`를 누르는 순간, "두 사람의 명리학적 기운을 분석하는 중..."이라는 세련된 스피너와 함께 분석 프로그레스 바(Progress Bar)가 먼저 시각적으로 보여야 리포트의 가치가 올라갑니다. 이 로딩 페이지 UX도 블루프린트에 포함할 것을 제안합니다.

## 🎨 3단계: UI/UX 및 화면 설계 (Wireframe Text)

- **배경색(Bg)**: `bg-[#0B071E]` (깊은 미드나잇 퍼플)

```
# 🔮 안티그래비티 개발 지시서: '연애판별소(연판소)' 메인 랜딩 페이지 구축

## 1. [Project Overview]
- **서비스명**: 연애판별소 (연판소)
- **핵심 상품**: 썸 단계에서 관계의 불확실성을 사주 데이터로 분석하는 '썸 손절 판별 리포트'
- **목적**: 애매한 관계에 놓인 사용자의 시간, 감정 낭비를 막고 확신 있는 연애 의사결정을 돕는 고전환(High-conversion) 랜딩 페이지 구축
- **핵심 타깃**: 썸 상대의 모호한 태도로 스트레스를 받으며, 모바일 환경에 극도로 친숙한 20대 여성 (Mobile-First UI/UX 필수)
- **메인 디자인 무드**: 기존의 칙칙한 사주 무드를 완전히 탈피하고, 현대적인 '글래스모피즘(Glassmorphism)'과 '미드나잇 오로라' 콘셉트를 결합한 신비롭고 세련된 대시보드 스타일

## 2. [Technical Stack & Libraries]
에이전트는 외부 자산 의존성을 최소화하고 브라우저에서 즉시 완벽하게 구동되도록 아래 Stack을 활용하여 단일 파일(`index.html`) 또는 구조화된 컴포넌트로 빌드하세요.
- **Framework & CSS**: HTML5, Tailwind CSS (CDN 연동)
- **Icons**: Lucide Icons (CDN) 또는 완벽히 스타일링된 인라인 SVG Code만 사용 (깨지는 이미지 링크 절대 금지)
- **Data Visualization**: Chart.js (CDN 연동) - 레이더 차트 및 도넛 차트 구현용
- **Animation**: Tailwind 내장 애니메이션 클래스 및 Custom CSS Keyframes를 활용한 마이크로 인터랙션

## 3. [Design System & Palette]
- **Color Palette**:
  - `Background`: Deep Midnight Purple (`bg-[#0B071E]`)를 베이스로 사용
  - `Card / Component`: 투명도와 블러가 적용된 White 계열 (`bg-white/5 backdrop-blur-md border border-white/10`)
  - `Primary / Point 1`: Neon Pink (`text-[#FF52A2]` / `bg-[#FF52A2]`) - 감성 및 주요 액션 유도
  - `Secondary / Point 2`: Neon Mint (`text-[#00F5D4]` / `bg-[#00F5D4]`) - 확신, 해답, 지수 표현
  - `Text`: Main White (`text-[#F3F4F6]`), Sub Mutted (`text-[#9CA3AF]`)
- **Typography**: 프리텐다드(Pretendard) 또는 Sans-serif 계열의 모던하고 깔끔한 폰트, 가독성 높은 자간/행간 유지
- **Shape Specs**: 주요 카드는 `rounded-2xl` (16px), 버튼은 `rounded-xl` (12px) 또는 `rounded-full` 적용. 부드러운 은은한 네온 글로우 효과(`shadow-[0_0_20px_rgba(255,82,162,0.2)]`) 적극 활용.

## 4. [Page & Visual Component Structure]
`index.html` 내에 아래 명세된 컴포넌트들을 순서대로 배치하고, 모바일 반응형(Grid/Flex 1열 전환)을 완벽히 구현하세요.

### ① Header Component (상단 네비게이션)
- 좌측에 `🔮 연애판별소` 로고 배치 (폰트 굵게, 네온 포인트 효과).
- 우측에 `[지금 판별하기 ⚡]` 버튼 고정 배치. Sticky top 속성을 부여하되, 배경에 `backdrop-blur-lg`를 주어 스크롤 시 콘텐츠와 자연스럽게 겹치도록 처리.

### ② Hero Section (상단 후킹 영역)
- **카피 구성**: 큰 타이틀로 *"이 썸, 계속 가야 할까? 지금 손절해야 할까?"*, 서브로 *"애매한 관계를 끝내고, 확신 있는 선택을 돕는 곳"* 배치.
- **백그라운드 비주얼**: CSS `animate-pulse`와 그라데이션 블러 필터를 조합하여 신비롭게 움직이는 보라색/핑크색 오로라 기운 표현.
- **CTA**: 대형 `[🔍 내 썸 상대 무료 성향 분석으로 시작하기]` 버튼 배치. 하단에 조그맣게 `(현재 2,341명이 판별 중)` 텍스트를 넣어 신뢰도 자극.

### ③ Problem & Empathy Section (페인 포인트 자극)
- "혹시 이런 '애매한 썸' 때문에 답답하신가요?" 타이틀 아래 3단 반응형 카드 배치.
- **Card 1 (메신저 UI)**: 스마트폰 대화창 스타일로 디자인. 상대방의 애매한 대화 예시("선톡은 오는데 만나자는 말이 없어요")를 말풍선 컴포넌트로 시각화. (Lucide `MessageCircle` 아이콘 활용)
- **Card 2 (타이밍 고민)**: 모던한 시계/모래시계 아이콘(Lucide `Clock`)과 함께 "계속 시간을 더 써야 하는 관계일까?" 텍스트 배치.
- **Card 3 (자존감 하락)**: 금이 간 하트 아이콘(Lucide `HeartCrack`)과 함께 "나 혼자 의미 부여하며 상처받는 기분" 텍스트 배치.

### ④ Interactive Report Preview Section (리포트 미리보기 - 핵심 기능)
- 실제 유저가 구매하게 될 '썸 손절 판별 리포트'의 핵심 대시보드를 인터랙티브하게 체험하는 영역.
- **좌측 (Chart 영역)**: 
  - **Chart A (Radar Chart)**: 관계 발전 가능성, 소통 지수, 성향 일치도, 미래 리스크 등을 매핑한 다각형 레이더 차트 렌더링.
  - **Chart B (Doughnut Chart)**: 중앙에 "손절 필요도 85%"가 네온 Mint 컬러로 크게 박힌 도넛 그래프 시각화.
- **우측 (상세 분석 판독서 영역)**:
  - 글래스모피즘 카드로 구성되어 있으며 다음과 같은 UI 뱃지와 텍스트 포함.
  - `• 상대방 연애 성향: 회피형 [Badge]` / `• 관계 흐름: 정체기` / `• 위험 신호: 🚨 Red Flag 감지!`
  - 하단 강조 박스: `👉 행동 가이드: "당분간 연락의 빈도를 30% 줄이고 상대의 반응을 살피세요."`
- **인터랙션**: 스크롤이 해당 섹션에 도달하면 Chart.js의 애니메이션 기능이 트리거되어 그래프 게이지가 부드럽게 차오르도록 스크립트 작성.

### ⑤ Input & Loading Section (사주 입력 및 트랜지션 효과)
- **입력 폼 UI**: '내 이름, 생년월일시(드롭다운)' 및 '상대 이름, 생년월일시(드롭다운)'를 입력할 수 있는 미래지향적 컴포넌트 설계. 필수 입력 미기재 후 버튼 클릭 시 폼 전체가 좌우로 흔들리는 `animate-shake` 예외 처리 구현.
- **로딩 시퀀스 (UX 핵심)**:
  - 사용자가 `[⚡ 썸 결과 판별하기]` 최종 버튼을 누르면, 입력 폼 영역이 부드럽게 `fade-out` 처리됨.
  - 동시에 그 자리에 **"🔮 두 사람의 명리학적 기운을 분석하는 중..."**이라는 메시지와 함께 로딩 스피너 및 0%에서 100%로 3초간 차오르는 프로그레스 바(`bg-[#FF52A2]`)가 `fade-in`으로 나타남.
  - 로딩이 100% 완료되면 "분석 완료! 리포트 생성 중..." 메시지와 함께 결과 확인/결제 유도 모달(또는 섹션)로 매끄럽게 전환되도록 JavaScript 가상 로직 구현.

### ⑥ Social Proof & Footer
- 인스타그램 피드 혹은 슬라이더 형태의 깔끔한 롤링 후기 컴포넌트 배치. (평점 별점 아이콘 포함)
- 푸터 영역에 20대 타깃이 안심할 수 있도록 이용약관, 개인정보처리방침, CS 안내 레이아웃 정돈.

## 5. [User Interaction & Animation Logic]
- **Hover Interaction**: 모든 주요 버튼(`Button`, `CTA`)은 마우스를 올렸을 때 `scale-105` 효과와 함께 배경의 그라데이션 색상이 부드럽게 반전되거나 밝아지는 `transition-all duration-300` 속성 부여.
- **Scroll Fade-In**: 테일윈드의 기본 인터랙션 또는 Intersection Observer API를 활용하여 사용자가 스크롤을 내릴 때 각 섹션(특히 Problem과 Report Preview)이 아래에서 위로 살짝 떠오르며 등장하는 애니메이션 적용.
```
